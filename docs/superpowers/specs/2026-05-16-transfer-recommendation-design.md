# 12306 AI 换乘推荐 — 设计文档

## 概述

为 12306-skill 新增 `transfer.mjs` 脚本，实现 AI 换乘推荐：BFS 广度搜索候选换乘路线 + LLM（OpenAI）根据用户偏好排序和解释。

## 动机

12306 官方换乘推荐的两个痛点：
1. 换乘站选择太少 — 只推荐大枢纽，忽略小站换乘可能性
2. 不支持同车换座 — 同一车次分段购票在中途换座位

## 脚本接口

```bash
node scripts/transfer.mjs <from> <to> [options]
```

| 参数 | 必需 | 说明 |
|------|------|------|
| `<from>` | 是 | 出发站或城市名 |
| `<to>` | 是 | 到达站或城市名 |
| `-d, --date` | 否 | 出行日期，默认今天 |
| `--max-transfers` | 否 | 最大换乘次数，默认 3 |
| `--min-transfer` | 否 | 最小换乘时间（分钟），默认 10 |
| `--preference` | 否 | 用户偏好描述，中文自然语言，传给 LLM |
| `-t, --type` | 否 | 车次类型筛选（G/D/Z/T/K，可组合） |
| `--seat` | 否 | 座位类型筛选（swz,zy,ze,rw,dw,yw,yz,wz） |
| `-f, --format` | 否 | 输出格式：md（默认）、html、json |
| `--model` | 否 | OpenAI 模型，默认 gpt-4o-mini |
| `--no-llm` | 否 | 跳过 LLM，仅输出 heuristic 排序结果 |

环境变量：`OPENAI_API_KEY`（LLM 排序必需）

## 搜索算法

### BFS 概述

从起点车站出发，逐层扩展可达车站，每层找到的车站作为下一层的起点，直到到达目的地或达到最大换乘次数。

### 候选站池

从 `stations.mjs` 的站点数据中，筛出地理上位于起点和终点之间的车站（按城市粗略过滤），避免查询明显在相反方向的车站。候选池在搜索前一次性计算。

> 注：station 数据不含经纬度，过滤基于站名/城市名的铁路走廊常识做方向判断，是 best-effort 启发式，允许多漏一些候选（宁可多漏不可错过滤）。

### 搜索流程

```
Layer 0: 查询 origin → 候选池中所有车站（并发，限流）
         记录所有有直达车的车站及其车次信息

Layer 1-N:
  对每层的每个活跃分支:
    1. 先查 当前站 → destination（目标导向）
       有结果 → 记录为完整路线，该分支终止
    2. 再查 当前站 → 候选池中其他车站（继续扩展）
  时间约束: 下段 depart > 上段 arrive + min_transfer_time
```

### 剪枝

- **目标导向**：每到达一个中间站，优先检查是否能直达终点，能则终止该分支
- **分支限制**：每层最多保留 50 个最有希望的分支（优先「能直达终点」，其次到达时间早的）
- **不走回头路**：同一路线中不重复出现同一车站
- **并发控制**：10-20 个并发请求，避免对 12306 造成压力

### 同车换座检测

BFS 自然结果：如果同一车次（trainCode）在一条路线的两段中出现（如 G123 北京南→济南西、G123 济南西→上海虹桥），自动标记为同车换座。

### 路线去重

相同车站序列（不考虑座位类型）的路线只保留最优的一条（按出发时间最早）。

## LLM 集成

### 调用时机

搜索完成后，取 heuristic 初筛的 top-30 候选路线送给 LLM 排序。

### Prompt 结构

**System**:
```
你是 12306 换乘推荐助手。根据用户偏好对候选路线排序，给出前 5 名推荐及简要理由。
偏好维度包括：总耗时、总价格、换乘次数、换乘时间充裕度、同站换乘、同车换座。
```

**User**:
```
用户偏好: {--preference 或默认「综合最优」}
出发: XXX 到达: YYY 日期: ZZZZ

候选路线 (JSON):
[
  {
    "rank": 1,
    "total_duration": "4h30m",
    "segments": [
      {"train": "G123", "from": "北京南", "to": "济南西", "depart": "08:00", "arrive": "09:30", "seats": {...}},
      ...
    ],
    "transfer_count": 1,
    "same_station_transfer": true,
    "same_train_seat_change": false,
    "min_transfer_time": 20
  },
  ...
]

请返回 JSON 数组，按推荐度排序，每项包含路线原始数据和 "reason" 字段（中文推荐理由）。
```

### LLM 返回格式

```json
[
  {
    "...路线原始字段...": "...",
    "reason": "总价最低 ¥523，济南西同站换乘仅需等待20分钟"
  },
  ...
]
```

### Fallback

LLM 调用失败时（网络错误、API key 无效）退回到 heuristic 排序结果，标注「（未使用 AI 排序）」。

### Markdown（默认）

```

偏好：省钱优先

| # | 方案 | 总耗时 | 换乘 | 换乘站 | 详情 | 推荐理由 |
|---|------|--------|------|--------|------|----------|
| 1 | G123→G456 | 4h30m | 1次 | 济南西 | 08:00-12:30 | 总价最低¥523，同站换乘... |
| 2 | ... | ... | ... | ... | ... | ... |

💡 方案 2 为同车换座：G789 北京南→上海虹桥，在济南西换座位即可
⚠ 数据来源 12306 · 2026-05-16 14:30
```

### HTML（`-f html`）

Apple 风格页面，参考 query.mjs 样式。增加换乘路线的层级展开展示，每段一个子行。

### JSON（`-f json`）

完整 LLM 排序后的候选列表，包含所有路线和每段列车的原始数据。
